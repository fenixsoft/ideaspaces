# 引言：线性代数 - 机器学习的语言

当我们谈论机器学习时，往往聚焦于复杂的算法模型和令人惊叹的应用场景，譬如 AlphaGo 击败围棋世界冠军、ChatGPT 生成流畅自然的对话、ADS 自动驾驶汽车在复杂的城市道路中穿行。我们都知道这是人工智能（Artificial Intelligence，AI）在背后驱动的结果，然而，深入本质，支撑这一切的底层语言其实是数学与逻辑，或者更具体地说，是**线性代数（Linear Algebra）**。

理解向量（Vector）和矩阵（Matrix）这些线性代数中的核心工具，不仅是对数学理论的探索，更是打开机器学习大门的钥匙。本章将从数据本质、几何视角和实际应用三个维度，阐述线性代数为何成为机器学习的核心语言。本系列文章定为是面为传统 IT 开发人员向机器学习、人工智能迈进所铺垫的前置知识，目的是讲清楚概念原理，并不是数学课程，因此不必担心繁琐的数学公式和计算过程，它们将会直接使用计算机（主要是 Python 语言和常用 Numpy 等计算工具库）替代。

## 空间中的箭头 - 向量

### 二维、三维空间的直观理解

线性代数的一个美妙之处在于它的**几何直观**。在二维平面上，向量可以理解为一个从原点出发的有向箭头。向量 $(3, 2)$ 表示从原点 $(0, 0)$ 指向点 $(3, 2)$ 的箭头。

![二维空间中的向量](./assets/vector_2d.png)

*图 1.1：二维空间中的向量 $\mathbf{v} = (3, 2)$，从原点指向坐标点 $(3, 2)$*

从几何上看，向量的加法有十分清晰的解释：$\mathbf{u} + \mathbf{v}$ 相当于先沿着 $\mathbf{u}$ 走，再沿着 $\mathbf{v}$ 走，最终在坐标轴中所到达的位置，这就是著名的**平行四边形法则**。

![向量加法的平行四边形法则](./assets/vector_addition.png)

*图 1.2：平行四边形法则——以 $\mathbf{u}$ 和 $\mathbf{v}$ 为邻边构造平行四边形，对角线即为向量和 $\mathbf{u} + \mathbf{v}$*

向量的乘法（内积，又称点积） 在几何上则代表着两个向量之间的夹角关系。当内积为零时，两个向量正交（垂直）；内积为正时，夹角为锐角；内积为负时，夹角为钝角。这里的不同的角度实际上度量了两个向量在方向上的“一致程度”。当两个向量指向相同方向时（$\theta = 0°$），余弦值为 1（$\cos\theta = 1$），内积达到最大值，表示它们高度相关；当两个向量垂直时（$\theta = 90°$），余弦值为 0（$\cos\theta = 0$），内积处于零值，表示它们相互独立、毫无关联；当两个向量方向相反时（$\theta = 180°$），余弦值为 -1（$\cos\theta = -1$），内积为负的最大值，表示它们呈负相关。这便是机器学习中使用向量的相似度来衡量文本、图像等数据相似性的数学基础——方向越接近的向量，语义越相似。

![向量内积的几何含义](./assets/vector_dot_product.png)

*图 1.3：向量内积的几何意义——从左到右分别为：同向（内积为正）、正交（内积为零）、反向（内积为负）*

### 以几何直觉辅助理解高维空间的抽象思维

线性代数的魅力在于几何与代数的完美对应，这使得人类可以通过几何直觉来与代数运算的对应，譬如以下表格中，使用向量的几何特征来理解数据的显示含义，要比死记硬背它的代数表示公式直观的多，这种对应让我们既能从几何直观理解概念，又能用代数方法精确计算。

| 几何概念 | 代数表示 | 现实含义 |
|---------|---------|---------|
| 向量相加 | $\mathbf{u} + \mathbf{v} = (u_1+v_1, \ldots, u_n+v_n)$ | 数据的叠加效果，如两个方向的合力、多个特征的综合影响 |
| 向量长度 | $\|\mathbf{v}\|_2 = \sqrt{v_1^2 + \cdots + v_n^2}$ | 数据的"规模"或"强度"，如信号的能量、向量的模长 |
| 向量夹角 | $\cos\theta = \frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{u}\| \|\mathbf{v}\|}$ | 数据间的相关性，夹角越小表示越相似，用于推荐系统和语义搜索 |
| 向量正交 | $\mathbf{u} \cdot \mathbf{v} = 0$ | 数据相互独立、毫无关联，如 PCA 中正交主成分代表不相关的特征 |
| 投影 | $\text{proj}_{\mathbf{v}} \mathbf{u} = \frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{v}\|^2} \mathbf{v}$ | 数据在某个方向上的"影子"，用于降维和特征提取 |
| 线性变换 | 矩阵乘法 $\mathbf{Ax}$ | 数据的转换、旋转或缩放，神经网络中的权重变换 |

面对更高维的空间，虽然我们无法直接“看见”四维、五维甚至更高维的空间，但线性代数的运算规则在任意维度都成立，这种抽象能力正是数学工具的强大之处。

在机器学习中，我们经常处理高维向量。譬如 GloVe 词向量通常是 300 维，BERT 的句向量可达 768 维甚至更高。虽然无法可视化，但我们仍然可以应用内积、变换、投影等运算来寻找向量之间的关系。

当然，高维空间也有一些反直觉的性质。譬如，在高维空间中，大多数点都分布在“边角”附近，而不是中心区域，这对理解某些机器学习算法的行为有重要启示。

## 数据的本质：一切皆向量

### 从原始数据到特征向量

机器学习的核心任务是让计算机从历史数据中学习到规律，然后应用于新的数据。那么，什么是数据？在机器学习的世界里，**数据就是向量**。

在计算机里，无论数据的原始形式如何，是图像、文本、音频、表格记录，或者其他别的形式。它们最终都会被转化为数值向量，然后才能被后续的各种机器学习算法处理。这个转化过程称为**特征提取（Feature Extraction）**。

来考虑一个简单的例子——房价预测：如果你要预测某套房子的价格，需要考虑哪些因素？可能包括房子的面积、卧室数量、卫生间数量、房龄、距地铁站距离，等等。这里每个因素就被称为一个**特征（Feature）**。将所有特征组合起来，就得到了一个特征向量：

$$\mathbf{x} = (\text{面积}, \text{卧室数}, \text{卫生间数}, \text{房龄}, \text{距地铁站距离})$$

对于一套 90 平米、两室一卫、房龄 5 年、距地铁站 500 米的房子，其特征向量可以表示为：

$$\mathbf{x} = (90, 2, 1, 5, 500)$$

这就是一个五维向量。一个用于预测房价的机器学习模型，实际上就是在这个五维空间中寻找规律，建立从特征向量 $\mathbf{x}$ 到房价 $\mathbf{y}$ 的映射关系。

### 不同类型数据的向量化表示

但是，现实世界的数据形式多种多样，并不总是直接与数字直接相关，譬如文字、声音、视频、图像，等等。它们如何转化为向量呢？

- **图像数据**：假设有一张 $28 \times 28$ 像素的 8 位灰度图像，它就可以表示为一个 784 维向量（$28 \times 28 = 784$）。每个维度对应一个像素点的灰度值（8 位图像具有 0-255 种灰度取值）。对于彩色图像，每个像素有 R、G、B 三个通道，因此一张 $28 \times 28$ 的彩色图像对应 $28 \times 28 \times 3 = 2352$ 维向量。

  ```python
  import numpy as np
  from PIL import Image

  # 加载图像并转换为向量
  image = Image.open('photo.jpg').convert('L')  # 转为灰度图
  image = image.resize((28, 28))  # 调整大小
  vector = np.array(image).flatten()  # 展平为 784 维向量
  print(f"向量维度: {vector.shape[0]}")  # 输出: 784
  ```

- **文本数据**：文本的向量化有多种方法。最简单的是**词袋模型（Bag of Words）**，它统计每个词出现的频次，形成频次向量。更先进的方法如 **Word2Vec**、**BERT** 能将单词或句子映射为语义丰富的稠密向量。

  ```python
  # 简化的词袋模型示例
  documents = ["深入理解 Java 虚拟机", "凤凰架构", "智慧的疆界"]
  vocabulary = set(" ".join(documents))  # 构建词汇表
  print(f"词汇表: {vocabulary}")

  # 每个文档转换为一个向量，维度等于词汇表大小
  # 向量的每个分量表示对应词的出现次数
  ```

- **声音数据**：音视频信号是时间序列，通过**傅里叶变换**或**梅尔频率倒谱系数（MFCC）** 等特征提取方法，可以转化为固定维度的特征向量。

### 向量空间模型在信息检索中的应用

向量不仅仅是数据表示的工具，同时也是数据计算的工具。譬如通过计算向量的相似度，来反映原始数据之间是否存在关联。在信息检索领域，**向量空间模型（Vector Space Model）** 是十分经典的方法。举个例子，假设你手上有一个文档库系统，希望实现根据用户输入信息进行语义检索，用户输入“华夏”，系统应当能把“中国”相关的文档找出来，你会如何实现？一种典型的做法是将每个文档表示为一个向量。当用户输入查询语句时，系统同样将查询转化为向量，然后计算查询向量与每个文档向量的相似度，返回最相似的文档，一种很常用的相似度度量是**余弦相似度（Cosine Similarity）**：

$$\text{similarity}(\mathbf{q}, \mathbf{d}) = \frac{\mathbf{q} \cdot \mathbf{d}}{\|\mathbf{q}\| \|\mathbf{d}\|}$$

其中 $\mathbf{q}$ 是查询向量，$\mathbf{d}$ 是文档向量。这个公式正是利用了向量的内积和范数来计算相似度。别担心，笔者此时拿出这个公式并不是让你背诵它，这里你只需要通俗地理解余弦相似度就好。前面提到过余弦相似度在几何上衡量的是**两个向量之间的夹角大小**，而非它们的长度，方向越相近的两个向量，语义越相似。

为什么用夹角而不是距离？因为文档长度不同会导致向量长度差异很大——一篇长文和一篇短文可能讨论相同的主题，但用词总量相差悬殊。如果用欧几里得距离（两点间的直线距离），长文档会因为向量更长而被判定为不相似。而余弦相似度**忽略了长度，只关注方向**，这样"机器学习入门"和"深入理解机器学习原理"这两个查询就能被正确匹配到相关文档，即使它们的篇幅不同。

再举个直观的例子：假设查询是"苹果价格"，文档 A 是"今日红富士苹果一斤 8 元"，文档 B 是"苹果公司发布新款 iPhone"。虽然 A 和 B 都包含"苹果"，但是：
- A 的向量方向指向"水果/农产品"方向
- B 的向量方向指向"科技公司/电子产品"方向

这两个方向差异较大（夹角接近 90°），余弦相似度就会很低，系统就能正确区分它们的不同语义。可以通过如下代码来实现这个文档库系统的简单原型：

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

documents = [
    "机器学习是人工智能的核心技术",
    "深度学习是机器学习的子领域",
    "自然语言处理处理文本数据"
]

# 将文档转换为 TF-IDF 向量
vectorizer = TfidfVectorizer()
doc_vectors = vectorizer.fit_transform(documents)

# 计算查询与文档的相似度
query = "机器学习"
query_vector = vectorizer.transform([query])
similarities = cosine_similarity(query_vector, doc_vectors)

print(f"查询: '{query}'")
for i, sim in enumerate(similarities[0]):
    print(f"文档{i+1} 相似度: {sim:.4f}")
```
