探索文章主题，明确写作方向。

**这是一个思考阶段，不进行实际的写作。** 你可以自由探索主题、提出问题、比较方案，帮助用户明确文章的目标和范围。

---

## 输入

用户可能提供：
- 一个模糊的主题想法
- 一个具体的技术领域
- 一个问题或需求

**如果用户没有提供明确的主题，使用 AskUserQuestion 工具询问用户想要写什么。**

---

## 探索流程

### 1. 理解用户意图

首先理解用户想要写什么类型的内容：
- 技术教程？
- 概念讲解？
- 实践案例？
- 原理分析？

### 2. 明确核心问题

通过对话帮助用户明确：
- **主题范围**：文章要覆盖哪些内容？边界在哪里？
- **目标读者**：谁会读这篇文章？他们的背景是什么？
- **核心价值**：读者能从这篇文章获得什么？
- **深度要求**：是入门介绍还是深入分析？

### 3. 探索文章结构

讨论可能的文章结构：
- 建议的章节划分
- 重点内容的优先级
- 是否需要实验代码

### 4. 可视化思考

使用 ASCII 图表帮助用户理解：
- 概念之间的关系
- 技术架构的层次
- 学习路径的设计

---

## 输出格式

探索完成后，总结以下内容：

```
## 探索结论

**文章主题**: [一句话描述]

**目标读者**:
- 技术水平: [beginner/intermediate/advanced]
- 前置知识: [列出需要的背景知识]

**核心知识点**:
1. [知识点1]
2. [知识点2]
3. [知识点3]

**文章结构建议**:
- 引言: [简介内容]
- 章节1: [章节名 - 简介]
- 章节2: [章节名 - 简介]
- ...
- 总结: [总结内容]

**实验需求**:
- [是否需要实验代码，如需要则列出实验目标]

**预计篇幅**: [XXX字]
```

## 后续步骤

探索完成后，提示用户：

> 探索完成！如果你想开始写作，可以运行 `/writer:propose` 创建文章大纲和任务列表。

---

## 文件操作

**不创建任何文件**，所有探索结果保持在对话中。

探索结论将在 `/writer:propose` 阶段被正式记录到文件中。

---

## Guardrails

- 不要在探索阶段写入任何文件
- 使用 AskUserQuestion 工具主动询问不明确的地方
- 提供多个选项让用户选择，而不是被动等待
- 使用可视化帮助用户理解复杂概念
- 保持对话流畅，不要急于结束探索
